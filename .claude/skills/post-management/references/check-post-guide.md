# 포스트 품질 체크리스트

모든 포스트는 아래 체크리스트를 **전부 통과**해야 한다. 검수/작성 시 항목별로 확인하고, 위반 사항은 즉시 수정한다.

## Checklist

### 1. Frontmatter

- [ ] 필수 필드 존재: `title`, `categories`, `tags`, `createdAt`
- [ ] 날짜 형식: ISO (`YYYY-MM-DDTHH:mm`)
- [ ] `categories`, `tags`가 배열 형태

### 2. 맞춤법 및 어법

- [ ] 띄어쓰기 정상 (예: "할수있다" → "할 수 있다")
- [ ] 맞춤법 정상 (예: "되" vs "돼", "안되" → "안 돼")
- [ ] 외래어 표기법 준수
- [ ] 조사 사용 정상 (예: "를" vs "을")
- [ ] 문장 종결 일관성
- [ ] 텍스트 화살표 → 유니코드 변환 (`->` → `→`, `<-` → `←`, `<->` → `↔`). 단, HTML 주석(`<!-- -->`) 제외
- [ ] **존댓말(경어체) 사용**
  - 금지: "~다.", "~한다.", "~된다.", "~이다.", "~있다.", "~없다.", "~않는다."
  - 사용: "~합니다.", "~됩니다.", "~입니다.", "~있습니다.", "~없습니다.", "~않습니다."
  - 해요체 혼용 허용: "~해요.", "~이에요.", "~거든요.", "~있어요.", "~죠.", "~세요."
  - 검사 대상: 본문 텍스트만 (코드 블록, frontmatter, 주석, Callout 내 인용문 제외)
- [ ] **콜론 패턴 금지**: 문장 끝 콜론(`:`)으로 코드 블록/목록을 소개하지 않음
  - 금지: "이렇게 작성하면:", "아래처럼 렌더링됩니다:", "다음과 같습니다:"
  - 허용: 콜론 없이 문장 종결 또는 마침표

### 3. 용어 통일

- [ ] 같은 개념의 표기가 통일됨 (예: "플러그인" vs "Plugin" vs "plugin")
- [ ] 한글/영문 표기 일관성 (예: "클로드 코드" vs "Claude Code")
- [ ] 대소문자 일관성 (예: "claude code" vs "Claude Code")
- [ ] 외래어 한글 음차 vs 영문 원어 혼용 없음 (예: "프리셋" vs "preset")
- [ ] 통일 기준이 불분명하면 사용자에게 질문

### 4. 구조

- [ ] heading 계층 순서 정상 (h2 → h3 → h4, 건너뛰기 없음)
- [ ] 빈 섹션 없음
- [ ] 모든 코드 블록에 언어 지정
- [ ] 이미지에 alt 텍스트 존재
- [ ] **구분선 사용 안 함** (`---`, `***`, `___` 금지. 섹션 구분은 heading으로)
- [ ] **코드 블록 `title` 속성 존재** (모든 코드 블록에 필수)
  - 1순위: 실제 파일명 (`title="plugins.ts"`)
  - 2순위: 의미 있는 이름 + 확장자 (`title="result.ts"`, `title="example.css"`)
  - 3순위: 언어 기반 파일명 (`title="script.ts"`, `title="style.css"`, `title="document.md"`, `title="text"`)

### 5. SEO

- [ ] `description` 필드 존재, 50~160자
- [ ] 적절한 키워드 포함
- [ ] 제목 60자 이하

## 검사 결과 출력

위반 사항 수정 후 아래 형식의 변경 내역 테이블을 출력한다.

```
| # | Line | Category | Before | After | Note |
|---|------|----------|--------|-------|------|
| 1 | 12 | 화살표 | -> | → | 유니코드 변환 |
| 2 | 25 | 맞춤법 | 안되 | 안 돼 | 띄어쓰기+어법 |
| 3 | 40 | 용어통일 | plugin | Plugin | 대소문자 통일 |
```

- **#**: 변경 순번
- **Line**: 라인 번호
- **Category**: 검사 항목 (화살표, 맞춤법, 용어통일, 구조, SEO 등)
- **Before / After**: 변경 전후
- **Note**: 사유
